DiagramaRelacional

Modelo Entidad-Relacion

  • Cliente
    • Atributos: ID_Cliente, Nombre, Dirección, Teléfono, Email
  • Producto (Propiedades)
    • Atributos: ID_Producto, Descripción, Precio, Stock, Estado (disponible, vendido, etc.)
  • Carrito de compras (Compra)
    • Atributos: ID_Carrito, Fecha, Total
  • Orden (Compra confirmada)
    • Atributos: ID_Orden, Fecha_Compra, Total, Estado (pagado, pendiente, cancelado)
  • Pago
    • Atributos: ID_Pago, Monto, Fecha_Pago, Método
  • Relaciones
    • Cliente - Carrito de compras: Un cliente puede tener uno o más carritos, pero generalmente uno activo.
    • Carrito de compras - Producto: Un carrito puede contener varios productos, y un producto puede estar en varios carritos (relación de muchos a muchos, con una tabla intermedia "DetalleCarrito").
    • Carrito de compras - Orden: Cuando el cliente realiza la compra, se genera una orden.
    • Orden - Pago: La orden se paga mediante un pago asociado.

Reglas de negocio

Cliente

  • Un cliente debe tener un nombre, dirección, teléfono y email válidos.
  • El email del cliente debe ser único.

Producto

  • Cada producto debe tener una descripción, un precio mayor a cero y un estado definido (disponible, vendido, etc.).
  • El stock debe ser un número entero igual o mayor a cero.
  • Un producto solo puede ser agregado al carrito si su estado es "disponible" y hay stock.

Carrito de Compras

  • Un carrito debe estar asociado a un cliente.
  • El total del carrito debe calcularse automáticamente en base a los productos y cantidades.
  • Un carrito no puede convertirse en orden si no tiene productos.

Orden

  • Una orden se genera a partir de un carrito confirmado.
  • La orden debe contener la fecha de compra, el total, y su estado (pagado, pendiente, cancelado).
  • Una orden no puede tener el estado "pagado" si no hay un pago asociado.

Pago

  • Cada pago debe estar vinculado a una orden.
  • El monto del pago debe coincidir con el total de la orden.

Relaciones Generales

  • La relación entre carrito y producto debe gestionarse mediante una tabla intermedia llamada "DetalleCarrito", que incluya cantidad, precio unitario y subtotal.
  • Un producto no puede eliminarse si está asociado a un carrito u orden.
  • Si se cancela una orden, el stock de los productos debe restablecerse.
